指令hexo d從本地佈署到github，傳送正常，卻無任何改變

造成原因

  1. 有成功傳送至Github，不過存在另一個分支，尚未合併=>需合併
  2. 在網址的導引上有誤=>setting修正
  3. 傳送可能依照網路與伺服器之反應速度有關，可能會有些許的延遲

解決方法

在_config.yml中的最下面deploy項目中branch所使用的名稱並沒有對到專案的名稱
正常流程應該是藉由 pull request 來達到檔案更新合併
倘若想直接上的話(內容確認無誤)，也可以將brach名稱改為main
這樣在使用 hexo d 後就不用再 pull request 了
也要記得如果有動到專案的setting需要回來確認是否正確
